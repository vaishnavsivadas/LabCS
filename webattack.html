Web Based Attack

What is Web-Based Attack ?

A web-based attack refers to malicious activities or techniques that target vulnerabilities and weaknesses in web applications, websites, or web servers. These attacks exploit security gaps in the software or configurations of these online platforms, aiming to compromise data, gain unauthorized access, disrupt services, or execute malicious code. These attacks can lead to data breaches, unauthorized access to sensitive information, financial losses, and disruption of online services.

What Is Cross-Site Scripting (XSS)?


Cross-Site Scripting (XSS) is a code injection attack in which an adversary inserts malicious code within a legitimate website. The code then launches as an infected script in the user’s web browser, enabling the attacker to steal sensitive information or impersonate the user.


Web forums, message boards, blogs, and other websites that allow users to post their content are the most susceptible to XSS attacks. Unless the input is reviewed, verified, and encoded by the web application, any malicious script included in the code will automatically run by other users’ browsers. This script can then access the user’s cookies, session tokens, or alternative sensitive information retained by the browser and used on that site. It is even possible for these scripts to rewrite the infected web page’s content in more advanced attacks.



How Does Cross Site Scripting Work?


It is important to note that XSS attack’s mechanics will vary based on the type of attack being deployed. That said, most attacks follow the same process:

Given that the code is not visible and most internet users do not understand common programming languages, such as JavaScript, it is difficult for the average user to detect XSS attacks.

What is Phishing ?


Phishing is when cybercriminals send fake messages or create fake websites that seem real to trick you into giving them your private information, like passwords or credit card details. They pretend to be trustworthy sources, like banks or companies, to steal your sensitive data and use it for bad purposes. Always be careful and verify before sharing personal info online.




Certainly! Here's an example of phishing:

Imagine you receive an email that looks like it's from your bank. The email claims that there's a security issue with your account and urges you to click on a link to update your login information to secure your account.

The email might say something like:

Dear Customer,


We have noticed some unusual activity on your bank account. To ensure the security of your account, please click on the link below to verify your information.

[Click here to verify your account]

Thank you for your prompt attention to this matter.

Sincerely,
Your Bank

However, if you carefully examine the email, you might notice some red flags:

1. **Generic Greeting:**The email starts with "Dear Customer" instead of addressing you by your name.

2. **Urgent Tone:** The email creates a sense of urgency, pressuring you to act quickly without thinking.

3. **Suspicious Link:** The link provided doesn't lead to the bank's official website. Hovering over the link (without clicking) might reveal a different website address.

4. **Unusual Sender Address:** The sender's email address might look slightly off or not match the bank's official domain.

If you were to click on the link and enter your login information, you could inadvertently provide your username and password to the attackers. They could then use this information to gain access to your bank account and potentially steal your money or commit other fraudulent activities. In this example, the attackers are using phishing tactics by posing as your bank to trick you into revealing sensitive information. It's important to always verify the authenticity of such emails by contacting the organization directly using contact information you find on their official website.

SQL Injection?


SQL injection is a type of cyber attack where malicious actors exploit vulnerabilities in an application's input validation to execute unauthorized SQL queries. It occurs when user inputs are not properly sanitized before being used in database queries, allowing attackers to manipulate these inputs to gain unauthorized access to the database or perform other malicious actions.




SQL injection is a code injection technique that might destroy your database.
SQL injection is one of the most common web hacking techniques.
SQL injection is the placement of malicious code in SQL statements, via web page input.



How SQL Injection Works?


Input:Web applications often take user inputs, like forms, search boxes, or URLs, and use them in SQL queries to interact with a database.

Lack of Sanitization:If the application doesn't properly validate or sanitize these inputs, attackers can inject malicious SQL code into them.

Manipulating Queries: Attackers insert SQL commands into input fields to manipulate the application's SQL queries. This can lead to unintended actions like unauthorized access, data leakage, or even modifying or deleting data.



Suppose a website has a login form:

SELECT * FROM users WHERE username = '<input_username>' AND password = '<input_password>';


If the application doesn't sanitize input correctly, an attacker could input ' OR '1'='1 as the username and ' OR '1'='1 as the password. The modified query would become:

SELECT * FROM users WHERE username = '' OR '1'='1' AND password = '' OR '1'='1';

SQL injection is a serious threat that can lead to data breaches and unauthorized access. Proper input validation, using prepared statements, and staying informed about secure coding practices are crucial for preventing these attacks.

Vulnerability?


A vulnerability in cybersecurity refers to a weakness or flaw in a system, network, application, or software that could potentially be exploited by malicious actors to compromise the security and integrity of that system. Vulnerabilities can range from programming errors and misconfigurations to design flaws and other weaknesses that could be leveraged to gain unauthorized access, steal sensitive information, disrupt services, or perform other malicious activities.




Nikto is a popular open-source web server scanner used to discover various vulnerabilities on web servers. It's often used for security assessments and penetration testing to identify potential weaknesses in web applications and server configurations.


In Kali Linux, Nikto is pre-installed and can be used from the command line. Here's a basic example of how to use Nikto:


Steps to perform nikto tool in Kali Linux

1. Open a terminal in Kali Linux.

2. Use the following command to scan a target website: Replace <target-website> with the actual URL of the website you want to scan.

nikto -h <target-website>


Nikto will start scanning the target website and display the results in the terminal. It will provide information about potential vulnerabilities, misconfigurations, and other security issues.
